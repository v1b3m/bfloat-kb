# obs-mcp

A lightweight MCP server that wraps the `obsidian` CLI, making all Obsidian vault operations available to any MCP-compatible AI client (Claude Code, Codex, Claude Desktop, etc.).

## Why

The `obsidian` CLI works great from a normal shell, but some AI clients (like Codex) run in sandboxed environments that can't invoke the CLI directly. MCP is a universal protocol — by wrapping the CLI in an MCP server, any client can interact with Obsidian without shell access.

## Architecture

```
AI Client (Claude Code, Codex, etc.)
  ↓ MCP protocol (stdio)
obs-mcp server (Node.js)
  ↓ child_process.execFile (reads) / fs.writeFile (writes)
obsidian CLI + vault files
```

- **28 tools** total: 19 CLI wrappers + 3 direct file I/O writers + 6 kanban tools
- Read operations (search, list, read) go through the CLI
- Write operations (create, append, prepend) use direct file I/O to avoid escaping issues
- Kanban tools parse and rewrite board markdown directly
- Transport: stdio

## Configuration

### Claude Code

MCP servers are configured per-project in `~/.claude.json` under the project key:

```json
{
  "projects": {
    "/Users/v1b3m/Dev/bfloat/bfloat-ide": {
      "mcpServers": {
        "obsidian": {
          "command": "node",
          "args": ["/Users/v1b3m/Dev/bfloat/obs-mcp/dist/index.js"]
        }
      }
    }
  }
}
```

To add it to other projects, add the same `obsidian` entry under that project's `mcpServers`.

### Claude Desktop

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "obsidian": {
      "command": "node",
      "args": ["/Users/v1b3m/Dev/bfloat/obs-mcp/dist/index.js"]
    }
  }
}
```

### Codex (OpenAI)

Add to `~/.codex/config.toml`:

```toml
[mcp_servers.obsidian]
args = ["/Users/v1b3m/Dev/bfloat/obs-mcp/dist/index.js"]
command = "node"
```

Codex uses TOML config and discovers MCP servers globally (not per-project).

### Other MCP clients

Same pattern — point to `node` with the `dist/index.js` path. Consult the client's docs for its MCP config format.

## Auto-approving tools

### Claude Code

By default, Claude Code prompts for permission on every MCP tool call. To auto-approve all obsidian tools globally, add to `~/.claude/settings.json`:

```json
{
  "permissions": {
    "allow": [
      "mcp__obsidian"
    ]
  }
}
```

This single rule covers all 28 tools (the prefix `mcp__obsidian` matches any tool from the server).

### Codex

Codex respects the global `approval_policy` in `~/.codex/config.toml`. If set to `"never"`, all MCP tools are auto-approved. There is no per-server approval config.
