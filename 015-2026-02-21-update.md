# 2026-02-21 Workbench Update Report

## Scope

This update focused on:
1. Stabilizing Expo dev-server restart behavior.
2. Preventing agent retry-loop false positives and improving max-turn handling.
3. Reducing log/memory pressure in desktop agent sessions.
4. Fixing font decode noise in Workbench UI.
5. Hardening Convex setup metadata synchronization.
6. Hardening Expo icon symbol fallback behavior and adding retrofit/validation tooling.
7. Updating Convex auth skill/templates for Expo vs Next.js correctness.

## Problems Observed
1. Restart race condition: restart commands were written while Expo was still stopping (`Stopping servercd ...`).
2. Infinite-loop guard blocked repeated successful commands in some scenarios.
3. `error_max_turns` surfaced as generic/unclear errors in UI and session handling.
4. Very noisy logging and oversized serialized payload logging in desktop provider increased memory pressure; Electron OOM observed.
5. Repeated `Failed to decode downloaded font` warnings from missing Workbench font assets.
6. Convex setup metadata/env values were not consistently reflected across backend response + Workbench state.
7. Expo-generated icon usage could fail on web/android when symbol names were unmapped.

## Changes Implemented

### 1) Deterministic restart and auto-recovery
Files:
- `packages/workbench/src/components/workbench/Workbench.tsx`
- `packages/workbench/src/lib/launch/index.ts`
- `apps/desktop/lib/mcp/terminal-mcp-server.ts`

What changed:
- Replaced fixed-delay restart behavior with staged logic.
- Added quick restart command builder (`buildDevCommand`) to avoid dependency reinstall on restart.
- Added restart concurrency guard.
- Added stop/start signal detection and timeouts.
- Added retry interrupt and terminal recreation fallback when stop cannot be confirmed.
- Updated restart MCP message to correctly describe quick restart behavior.

Result:
- Restart now succeeds reliably and avoids prior command concatenation race.

### 2) Agent provider robustness and log reduction
File:
- `apps/desktop/lib/agents/providers/claude-provider.ts`

What changed:
- Retry-loop guard now blocks repeated failed Bash retries (not repeated successful commands).
- Added failure summarization for repeated-failure abort messages.
- Added bounded auto-continue behavior for `error_max_turns`.
- Mapped SDK max-turn result to clearer error semantics.
- Reduced logging volume and payload size:
  - compact SDK options logging
  - truncated long stderr/text previews
  - removed noisy per-message debug chatter

Result:
- Lower false aborts, better error clarity, reduced log/memory pressure.
