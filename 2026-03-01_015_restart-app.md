### Context

the problem with the current approach is it stops the current server and immediately tries to start up another. This will fail often because the terminal and/or other processes are not ready immediately. A slight delay mechanism can save us this pain.

### Acceptance Criteria

- [x] The server restarts without failure most of the time, it currently fails to start most of the time
- [x] Restart waits for terminal/process interruption to settle before sending the next start command

### Constraints

- None
